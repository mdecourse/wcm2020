利用 openssl 建立 self-signed key 與 certificate

在 Windows 可以利用命令列中的 where openssl 查詢所使用的 openssl 指令位置

openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout localhost.key -out localhost.crt

req  - request

-x509 x509

-nodes - no DES (not to encrypt the private key with DES)

-days - good for 365 days

-newkey

rsa:2048

-keyout

-out outfile - output to filename rather than stdout

key 通常指 private key

csr certificate signing request

crt certificate

pem privacy-enhanced mail

有關 Python 程式中的 __name__

最簡單的 Flask 程式:

from flask import Flask
import ssl

# 即使在近端仍希望以 https 模式下執行
context = ssl.SSLContext(ssl.PROTOCOL_TLSv1_2)
context.load_cert_chain('localhost.crt', 'localhost.key')

app = Flask(__name__)

uwsgi = False

@app.route('/')
def hello_world():
    return 'Hello, World!'
    

if uwsgi:
    # 表示程式在雲端執行
    application = flaskapp.app
else:
    # 表示在近端執行, 以 python3 wsgi.py 執行
    flaskapp.app.run(host='127.0.0.1', port=6443, debug=True, ssl_context=context)

當 Flask 程式以 uwsgi 模式執行, SSL 的認證流程可以透過 NGINX 達成, 網站所屬的正式數位簽章則可以透過 https://letsencrypt.org/ 取得.
